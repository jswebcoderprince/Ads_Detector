
# Changelog

All notable changes to the Ads Detector project will be documented in this file.

This project follows a simple versioning strategy inspired by Keep a Changelog.

---

# Versioning Format

Major.Minor.Patch

Example:

1.0.0

Where:

- Major = Breaking changes
- Minor = New features
- Patch = Bug fixes and documentation updates

---

# Release Types

## Added

New features added to the project.

---

## Changed

Existing functionality has been modified.

---

## Improved

Performance, usability, or internal improvements.

---

## Fixed

Bug fixes.

---

## Removed

Features removed from the project.

---

## Security

Security-related improvements.

# Update Rules

Whenever a new feature is completed:

1. Increase the version number if required.
2. Add the release date.
3. Document all significant changes.
4. Keep entries in reverse chronological order.

---

# Example Entry

## [1.0.1] - YYYY-MM-DD

- Improved timeout handling.

# Notes

Small internal refactoring does not require a version bump unless it affects functionality.

Documentation-only updates may use a Patch version.
